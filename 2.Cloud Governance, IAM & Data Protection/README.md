# 🔐 Cloud Governance, IAM & Data Protection

**PT-BR:**

Este repositório demonstra a aplicação de práticas de **Governança em Nuvem, Controle de Acesso e Proteção de Dados** em ambientes AWS.

Os projetos apresentados exploram conceitos como **Identity and Access Management (IAM), Least Privilege, controle de permissões, criptografia, monitoramento e conformidade**, demonstrando como controles técnicos apoiam a Governança de Dados em ambientes modernos.

O objetivo é mostrar como políticas de acesso, segurança e governança contribuem para garantir que dados corporativos sejam utilizados de forma segura, controlada, rastreável e alinhada aos princípios de privacidade.


**EN:**

This repository demonstrates the application of **Cloud Governance, Access Control, and Data Protection practices** within AWS environments.

The projects explore concepts such as **Identity and Access Management (IAM), Least Privilege, access policies, encryption, monitoring, and compliance controls**, showing how technical safeguards support Data Governance in modern environments.

The goal is to demonstrate how security and governance controls ensure that enterprise data is accessed, protected, and managed in a secure, traceable, and privacy-aligned manner..

### 🛠️ Technical Enablers / Technical Pillars:

* **Identity & Access Governance:** Implementation of IAM principles, RBAC (Role-Based Access Control), least privilege policies, and access auditing to ensure controlled and traceable data access.

* **Cloud Governance & Infrastructure Automation:** Application of Infrastructure as Code (IaC) using Terraform to create standardized, controlled, and compliant AWS environments.

* **Data Protection & Privacy Controls:** Implementation of encryption practices, AWS KMS key management, and security controls to protect sensitive data throughout its lifecycle.

* **Compliance & Security Governance:** Application of governance principles, monitoring practices, and control frameworks to support secure and auditable cloud data environments.
